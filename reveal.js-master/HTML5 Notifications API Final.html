Notifications

API für Benachrichtigungen der End-Nutzer

... damit können Ereignisse bzw. Vorgänge gemeldet werden

... alamiert Nutzer außerhalb des Kontexts einer Webseite.

Die Notifications API ist eine Schnittstelle um den End-Nutzer über bestimmte Ereignisse bzw Vorgänge, die ihn betreffen, zu benachrichtigen. Dafür wird der Nutzer außerhalb des Kontexts einer Webseite alamiert....Beispiel...

Beispiele

  • Eingang einer eMail
  • Erinnerung an einen Termin
  • Etc.
Beispiele dafür sind unter anderen der Eingang einer eMail oder auch die Erinnerung an einen Termin oder an einen Geburtstag.

Darstellung

Meldungen können in verschiedenen Formen/Plätzen auftreten:

  • Ecke des Displays
  • im Bereich des Chromes des Nutzerbevollmächtigten
  • auf dem Home-Bildschirm eines mobilen Gerätes
Dargestellt werden die Notifications oftmals in der Ecke des Displays, wie man an dem Beispiel gerade eben sehen konnte. Aber auch im Bereich des Chromes des Nutzers oder auf dem Home-Bildschirm eines mobilen Gerätes können die Notifications angezeigt werden. zB wird auf meinem Handy jeden Tag per Benachrichtigung angezeigt wer an dem Tag Geburtstag hat.

Darstellung

Die API reagiert flexibel auf die Präsentationsoptionen des Nutzers.

Diese Spezifikation definiert jedoch nicht, wie der Nutzerbevollmächtigte die Benachrichtigung angezeigt haben möchte, denn die API ist entwickelt wurden, um flexibel auf die Präsentationsoptionen zu reagieren.

Konstruktor

Notification.Notification()

Aufgerufen werden die Benachrichtigungen durch den Kontruktor Notification.Notification().

Eigenschaften

Jede Benachrichtigung hat: Titel, Richtung, Sprache, Herkunft

Jede Benachrichtigung kann haben: Körper, ID, Symbol URL, Symbol Bild

Die Eigenschaften einer Benachrichtigung können in statische Eigenschaften und instanzierte Eigenschaften unterschieden werden. Jede Notification beinhaltet auf jeden Fall einen Titel, eine Richtung, eine definierte Sprache und eine Herkunft. Aber auch ein Körper, eine ID und ein Symbol mit der dazugehörigen URL und dem Bild könnten angezeigt werden.

Statische Eigenschaften


Notification.permission

  • default (Nichterfüllung)
  • denied (Verweigerung)
  • granted (Genehmigung)
Benachrichtigungen können nur angezeigt werden, wenn der Nutzer seine Erlaubnis dazu gegeben hat. Da gibt es drei Eigenschaften: default (Nichterfüllung), denied (Verweigerung) und granted (Genehmigung).

Instanzierte Eigenschaften


Notification.title

Notification.dir

Notification.lang

Zu den instanzierten Eigenschaften gehören der Titel und der Text der Notification, sowie die Sprache.

Instanzierte Eigenschaften


Notification.body

Notification.tag

Notification.icon

Aber auch der Körper, die ID und das URL des Bildes, welches als Symbol verwendet werden kann, können in einer Notification eingefügt werden.

Instanzierte Eigenschaften


Notification.data

Notification.silent

Mit ... kann eine strukturierte Kopie der Daten wiedergegeben werden und mit ... kann spezifiert werden, ob die Benachrichtigung leise, daher ohne Klang oder Vibration widergegeben werden soll.

Methoden


Notification.requestPermission()

Notification.close()

Auch die Methoden einer Benachrichtigung können in statische Eigenschaften und instanzierte Methoden unterschieden werden. Zu den statischen Methoden gehört ..., die sich die Erlaubnis des Nutzers, die Benachrichtigung anzuzeigen, holt. Mit ... kann das Programm eine Benachrichtigung automatisch schließen.

Ablauf

Schritt 1

Überprüfung ob Browser Notifications unterstützt

  • Nein: alert "nicht unterstützt"
  • Ja: Schritt 2
Der Ablauf einer Notification kann in drei Schritte unterteilt werden. Im 1. Schritt wird überprüft, ob der Browser die Notifications unterstützt. Wenn das nicht der Fall ist, gibt es eine Warnung. Ansonsten geht es weiter zu Schritt 2.

Ablauf

Schritt 2

Überprüfung ob die Erlaubnis für Notifications gewährt ist

  • Nein: um Erlaubnis fragen
    • wenn akzeptiert: Schritt 3
  • Ja: Schritt 3
Hier wird nun überprüft ob es eine gewährleistete Erlaubnis für Notifications gibt. Wenn nicht, wird der Nutzer um Erlaubnis gefragt (Bild?!). Sobald eine Erlaubnis vorhanden ist, geht es weiter zu Schritt 3.

Ablauf

Schritt 3

Erstellung der Notification

Hier, im letzten Schritt, wird die Notification erstellt.
